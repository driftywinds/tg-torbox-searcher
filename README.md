# torbox-tg-bot

> **⚠️ Disclaimer:** This entire project was generated by an AI assistant (Buffy, Freebuff's coding agent) based on the TorBox API documentation and user requirements. While care has been taken to avoid leaking API keys or tokens in logs and error messages, the user assumes all risk when using this bot. Review the code before exposing it to untrusted users. See the [security notes](#security--privacy) below for details on what is and isn't logged.

A Telegram bot that searches torrents via a **Jackett** (Torznab) instance, cross-checks results against **TorBox's** cache, lets a user select a result, and uploads it to **GoFile.io** via TorBox's integration API — all through simple numbered-list replies (no cluttered inline keyboards).

## What it does

1. **Authorized user sends free text** → bot queries your Jackett instance (Torznab aggregate endpoint), then cross-checks every result's hash against TorBox's `/torrents/checkcached` endpoint. Replies with a numbered, **non-interactive** list: name, size, cached ✅/⏳, tracker, seeders.

2. **User replies with a number** (e.g. `3`):
   - **Cached** ✅ → bot adds it to TorBox (`/torrents/createtorrent`), queues a GoFile upload via TorBox's integration API (`POST /integration/gofile`), polls until complete, and sends you the GoFile link.
   - **Not cached** ⏳ → bot starts the TorBox download and tells you to reply with the same number later to check progress. Once finished it automatically moves into the GoFile upload step.

3. **Admin commands** (admin ID set in `.env`):
   - `/adduser <telegram_user_id>` — authorize a new user
   - `/removeuser <telegram_user_id>` — revoke access
   - `/listusers` — show all authorized users

   Authorized users are persisted to `DATA_DIR/authorized_users.json`.

## Search backend

The bot uses **Jackett** as its search backend via the [Torznab](https://github.com/Sonarr/Sonarr/wiki/Implementing-a-Torznab-indexer) protocol (XML/RSS). This avoids TorBox's own search API which is IP-restricted to whitelisted partners (ElfHosted, etc.). The flow:

```
User query → Jackett (Torznab aggregate) → magnets/hashes
           → TorBox /checkcached (cache status) ✓
           → TorBox /createtorrent (add download) ✓
           → TorBox integration → GoFile upload ✓
```

You can also point it at **Prowlarr** or any other Torznab-compatible indexer by changing the `TORZNAB_URL`.

## Security & privacy

- **API keys in logs:** The bot masks API keys in terminal log output (shown as `***`).
- **API keys in Telegram messages:** Error messages sent to Telegram **do not** contain raw API keys — they are sanitised before being sent.
- **No webhook / public ports:** The bot uses Telegram long-polling — no inbound ports needed.
- **Data stored:** Only the list of authorized Telegram user IDs (plain JSON file in `DATA_DIR/`).

## Requirements

- Go 1.21+ (for local builds) **or** Docker
- A **Telegram bot token** (message [@BotFather](https://t.me/BotFather), `/newbot`)
- A **TorBox account** + API key (torbox.app → Settings → API key)
- A **GoFile.io API token** (gofile.io → myProfile → API Token) — used as `gofile_token` in TorBox's integration
- A **Jackett** (or Prowlarr) instance with at least one indexer configured
- Your **Telegram numeric user ID** (message [@userinfobot](https://t.me/userinfobot))

## Quick start

### Local (Windows / Linux / macOS)

```bash
cp .env.example .env
# Edit .env with your tokens (see .env.example for all options)

go build -o torbox-bot .
./torbox-bot
```

### Docker

```bash
cp .env.example .env
# Edit .env with your tokens

docker compose up -d --build
docker compose logs -f
```

## Env configuration

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `TELEGRAM_BOT_TOKEN` | ✅ | — | From @BotFather |
| `TORBOX_API_KEY` | ✅ | — | From torbox.app/settings |
| `GOFILE_API_KEY` | ✅ | — | GoFile Account API Token (gofile.io/myProfile) |
| `ADMIN_TELEGRAM_ID` | ✅ | — | Your numeric Telegram user ID |
| `DATA_DIR` | ❌ | `./data` | Where to persist authorized-users.json |
| `SEARCH_BACKEND` | ❌ | `torbox` | `torznab` (recommended) or `torbox` |
| `TORZNAB_URL` | if `torznab` | — | Your Jackett/Prowlarr base URL |
| `TORZNAB_API_KEY` | if `torznab` | — | Your Jackett/Prowlarr API key |
| `TORBOX_API_BASE_URL` | ❌ | `https://api.torbox.app/v1/api` | Only if TorBox changes this |
| `TORBOX_SEARCH_BASE_URL` | ❌ | `https://search-api.torbox.app` | Only if TorBox changes this |

## Testing it

1. Open a DM with your bot on Telegram and send `/start`.
2. Send a search query: `ubuntu 24.04`
3. Reply with a number to select a result.
4. If cached, the bot will upload to GoFile and send you the link. If not, reply with the same number later to check progress.
